In this lab, we will develop neural network models to predict whether car accidents result in injuries/fatalities (a classification task) and to predict the price of used Toyota Corollas (a prediction task).

We will need the following packages for this lab:

library(neuralnet)
library(nnet)
library(caret)

Neural Net for Classification

In the accidentsnn.csv file, U.S. automobile accidents have been classified by their level of severity as no injury, injury, or fatality. It may be worthwhile to develop a system to quickly classify the severity of an accident based on initial reports and associated data in order to appropriately assign emergency response team priorities. The dataset contains the following variables:

  • ALCHL_I: the presence (1) or absense(2) of alcohol
  • PROFIL_I_R: profile of the roadway - level (1) or other (0)
  • SUR_COND: surface condition of the road - dry (1), wet (2), snow/slush (3), ice (4), or unknown (9)
  • VEH_INVL: the number of vehicles involved
  • MAX_SEV_IR: the presence of injuries/fatalities - no injuries (0), injury (1), or fatality (2)

We can easily convert ALCHL_I to a binary dummy variable using an ifelse() function.

Training the Model

The data is now ready to be used in building a neural net model. We will begin by creating a model with a single hidden layer containing two nodes.

# run nn with 2 hidden nodes
# use hidden= with a vector of integers specifying number of nodes in each hidden layer
nn <- neuralnet(MAX_SEV_IR ~ ., data = acc.train, hidden = 2)

Plotting the Network

# plot the network
plot(nn, rep = "best")

We can see in the plot that the total error for this tree is 60.232798, and it required 5,766 iterations through the data to satisfy the stopping criteria.

Neural nets performance can be negatively affected by highly skewed predictors, so a log transformation should be applied to highly skewed predictors prior to building any models. Highly skewed predictors can be identified using box plots and/or the skewness() function in the e1071 package.

Here we look at all the numerical predictors in toyota.sub.

library(e1071)
num.cols <- colnames(toyota.sub[, c(2,4,6:7,9)])
for (i in num.cols) {
  boxplot(toyota.sub[[i]],
          ylab = i)
  print(skewness(toyota.sub[[i]]))
}

Generally, a variable is considered highly skewed if the absolute value of the skewness is greater than 1. In this case, we see that Quarterly_Tax and Guarantee_Period are highly skewed, so we apply a log transformation.

toyota.sub$Quarterly_Tax <- log(toyota.sub$Quarterly_Tax + 1)
toyota.sub$Guarantee_Period <- log(toyota.sub$Guarantee_Period + 1)

(We actually take the natural log of the variable plus 1 in order to avoid errors when the variables value is 0.)

Training the Model

We will begin by training a neural net model with a single hidden layer containing two nodes.

nn1 <- neuralnet(Price ~ ., data = toyota.train, linear.output = FALSE, hidden = 2)

Lets take a look at the predicted price for the first record in the training dataset.

nn1$response[1, ]
## [1] 0.5417407

Recall that when predicting a numerical/continuous outcome, you will need to rescale the output to its original units by multiplying by the original range and adding the original minimum.

nn1$response[1, ]*(max(toyota.df$Price) - min(toyota.df$Price)) + min(toyota.df$Price)
## [1] 19600

Plotting the Network

plot(nn1, rep = "best")

Assessing the Predictive Performance

As with other prediction algorithms, we will use root mean square error (RMSE) to assess the predictive performance of the model. We use the compute() function to calculate the predicted price based on the predictors and then use the RMSE() function to calculate RMSE.

In-Sample:

predict1.train <- compute(nn1, toyota.train[, c(2:16)])
RMSE(predict1.train$net.result, toyota.train$Price)
## [1] 0.03961706

Out-of-Sample:

predict1.valid <- compute(nn1, toyota.valid[, c(2:16)])
RMSE(predict1.valid$net.result, toyota.valid$Price)
## [1] 0.04682752

Summary

  • Neural networks perform very well for both classification and prediction tasks.
  • Neural nets capture very complex relationships among predictors and outcome variables.
  • Neural nets are black box in nature making the results difficult to interpret.
  • All variables should be standardized to a [0, 1] scale prior to inputting them into the model.
  • Categorical variables must be converted to binary dummy variables.
  • For predicting a continuous outcome, the output value must be rescaled to its original units.
