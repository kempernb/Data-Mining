Association Rules

Association rules is a popular and well-researched method for discovering interesting relationships between variables in large databases. The arules package in R provides a basic infrastructure for creating and manipulating input datasets for analyzing the resulting item sets and rules.

  • For and introduction to arules and additional case studies, see here.
  • For the reference manual for the package, see here.

The Groceries dataset (included in the arules package) contains one month (30 days) of real-world point-of-sale transaction data from a typical local grocery outlet. The dataset contains 9,835 transactions, and the items are aggregated to 169 cateories.

Recall from class:

  • Support: The support of an item set X is defined as the proportion of transactions in the dataset which contain the item set. In the Groceries data, the support for the rules is relatively low, with a maximum support of less than 3%.
  • Confidence: The confidence of a rule \(X \implies Y\) (“If X, then Y”) is defined as \(conf(X \implies Y) = supp(X \cup Y) / supp(X)\). For example, if the rule \({milk, bread} \implies {butter}\) has a confidence of 0.5, this means that for 50% of the transactions containing milk and bread, the rule is correct. Confidence can be interpreted as an estimate of the conditional probability \(P(Y | X)\), the probability of finding the consequent (RHS) of the rule in transactions under the condition that the transactions also contain the antecedent (LHS). Association rules are also required to satisfy both a minimum support and a minimum confidence constraint simultaneously.
  • Lift: Lift is a popular measure to filter or rank found rules. The lift of a rule \(X \implies Y\) is defined as \(lift(X \implies Y) = supp(X \cup Y) / (supp(X)supp(Y))\). Lift can be interpreted as the deviation of the support of the whole rule from the support expected under independence given the supports of the antecedent and consequent. Greater lift values indicate stronger associations.

Clustering

seed <- read.table("seeds_dataset.txt")

Let’s see if we can determine a number of clusters that shows better distinction among clusters.

Determine the Number of Clusters

Here, a simple within-group sum of squares method (wss) is used to determine the appropriate number of clusters. We can calculate the wss for different numbers of clusters.

wss <- rep(0, 12)
for (i in 1:12) {
  wss[i] <- sum(kmeans(seed, centers = i)$withinss)
}
plot(1:12, wss, type = "b", xlab = "Number of Clusters", ylab = "Within group sum of squares")

One method for choosing the optimal number of clusters is called “the elbow method”. If the line chart looks like an arm, then the “elbow” on the arm is the value of k that is the best. The idea is that we want a small SSE, but that the SSE tends to decrease toward 0 as we increase k (the SSE is 0 when k is equal to the number of data points in the dataset, because then each data point is its own cluster, and there is no error between it and the center of its cluster). So our goal is to choose a small value of k that still has a low SSE, and the elbow usually represents where we start to have diminishing returns by increasing k.

Summary

  • Association rules are useful for determining “what goes with what” - can be great tools for creating sales promotions, identifying comorbidities in healthcare, etc.
  • Ideally, you want to find rules with high support, confidence, and lift.
  • Clustering is an exploratory task for finding groups of similar records.
  • It is critical that your clusters are meaningful and not just randomly assigned.
  • Upon finding useful clusters, cluster membership can become a predictor in any supervised learning task you attempt.
